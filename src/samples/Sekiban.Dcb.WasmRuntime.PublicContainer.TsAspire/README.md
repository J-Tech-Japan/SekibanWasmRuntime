# Public Container TypeScript AppHost Sample

Hosts the **public** Sekiban WASM Runtime container
(`ghcr.io/j-tech-japan/sekiban-wasm-runtime-host:1.0.0-preview.3`) from an
**Aspire TypeScript AppHost** (`apphost.mts`) using the thin
[`@sekiban/aspire`](../../lib/sekiban-aspire-ts) helper, with Postgres for the
event store and materialized views.

The wasm module + manifest come from the CsDecider public-container sample —
the hosting model is language-agnostic. Build them first, then run the smoke:

```bash
bash src/samples/Sekiban.Dcb.WasmRuntime.PublicContainer.CsDecider/scripts/build-wasm.sh
bash src/samples/Sekiban.Dcb.WasmRuntime.PublicContainer.TsAspire/scripts/smoke.sh
```

The smoke packs `@sekiban/aspire`, installs the sample against the packed
tarball (`--no-save`, so `package.json` keeps the dev-time `file:` reference),
starts the AppHost headlessly with `aspire run`, and validates health/ready
plus commit → tag-latest-sortable → list-query against the public container.

Prerequisites: Docker, Node.js 20+, Aspire CLI 13.4+. `.aspire/modules` is
generated by `aspire run` (not committed); the PostgreSQL integration is
recorded in `aspire.config.json`.

Interactive run: `cd` into this directory and `aspire run` (after `npm
install`), then open the dashboard it prints.

See [`docs/samples/public-container-ts-apphost.md`](../../../docs/samples/public-container-ts-apphost.md)
for the comparison with the C# `Sekiban.Dcb.WasmRuntime.Aspire` package.
