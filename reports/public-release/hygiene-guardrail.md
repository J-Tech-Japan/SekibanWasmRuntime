# SWR-G004 Public Repository Hygiene Guardrail

## Summary

The public repository hygiene check is now tracked in `scripts/check-public-hygiene.sh`.
It inspects only files already tracked by git, so local ignored files such as `.takt/`
or editor state do not affect the result.

## Guarded host-local and generated artifacts

- Unclassified host-local automation/editor state must not be tracked: `.takt/`,
  `.intent-cli/`, `.codex/`, `.idea/`, and local `.claude/` or `.vscode/`
  state outside the classified repository guidance/config files.
- User and OS local files must not be tracked: `.DS_Store`, `*.user`, `*.suo`,
  `*.rsuser`, and related solution user-state files.
- Build and dependency caches must not be tracked: `bin/`, `obj/`, `node_modules/`,
  `.next/`, `.generated/`, `target/`, `_build/`, `.mooncakes/`,
  `BenchmarkDotNet.Artifacts/`, `TestResults/`, and `playwright-report/`.
- Generated benchmark `.log` files under `benchmarks/results/` are no longer
  tracked. Structured benchmark result `.json` files and
  `benchmarks/results/BENCHMARK-RESULTS.md` remain tracked as public evidence.
- Generated AppHost runtime manifests under `src/samples/**/.generated/` are no
  longer tracked. They can contain machine-specific absolute module paths and are
  regenerated by the sample AppHost projects.

## Intentionally retained generated or vendored files

- `src/lib/sekiban-ts/dist/` remains tracked as the current TypeScript package
  distribution output. This is classified as package output, not host-local state.
- `src/samples/**/wwwroot/lib/bootstrap/dist/` remains tracked as sample static
  web vendor assets required by the current sample layout.
- `src/**/modules/*.wasm` remains tracked where it is sample or internal usage
  source material. Binary module packaging policy is handled separately by
  SWR-G005, so these files are classified rather than removed here.
- `.claude/skills/` and `.vscode/mcp.json` remain tracked as repository guidance
  and tool configuration. Local state remains ignored by `.gitignore`.

## Verification

Run:

```bash
bash scripts/check-public-hygiene.sh
git ls-files '.takt/*' 'benchmarks/results/*.log' 'src/samples/**/.generated/**'
git diff --check
```

Expected result:

- `scripts/check-public-hygiene.sh` reports `public hygiene check passed`.
- `git ls-files '.takt/*' 'benchmarks/results/*.log' 'src/samples/**/.generated/**'`
  prints no tracked files.
- `git diff --check` reports no whitespace errors.
